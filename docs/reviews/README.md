# Review-Reports

Uebergabe-Artefakte Reviewer → Implementation. Ein Report pro Lauf,
Dateiname `<YYYY-MM-DD>-<slice-oder-diff-ref>.md`; Folgelaeufe
bekommen eine neue Datei, keine Ueberschreibung (Auditierbarkeit).

Vorlage: `lab/templates/docs/reviews/review-report.template.md` im
Kurs-Repo (siehe [Konventionen](../../harness/conventions.md),
Abschnitt Adoptierte Konventions-Quellen).

Noch keine Reports vorhanden — der erste entsteht mit dem ersten
Implementierungs-Slice.
